CardsViewController

The CardsViewController uses a collection view where each cell is a single card. The cells will render the frames for the cards, but are designed to have a child ViewController that will display the contents of the card. Thus CardsViewController is essentially a list of child view controllers that are rendered in special cells that draw the card frames. To use a CardsViewController, you first need to subclass it. Then in the viewDidLoad method call the super class loadCards method passing an array of CardControllers. Each instance of a CardController will be rendered as a single card. The loadCards method does this by getting the view controller for each CardController and adding them as child view controllers to the card cells.

Example:

 class TestCardsViewController: CardsViewController {

    let cards: [CardController] = [TestCardController(), AnotherTestCardController()]

    override func viewDidLoad() {
        super.viewDidLoad()

        loadCards(cards: cards)
    }
}

    Sets up the collection view:

    • Uses automatic sizing for collection view cells (if >iOS 10)
    • .vertical scroll direction
    • backgroundColor, insets
    • Pinning constraints, CardParts theme spacing

  • invalidateLayout()

    Invalidate layout to redraw

  • loadCards(cards:)

    Sets internal cardControllers to cards and registers cells for each card. Reloads data and invalidates layout.

  • registerCells(cards:)

    Registers each card to the collectionView by its hashString

  • reload(cards:)

    Calls loadCards

  • collectionView(_:numberOfItemsInSection:)

    Returns number of card conterollers

  • collectionView(_:cellForItemAt:)

    Dequeues collection view cell. Constructs and applies delegates/protocols as needed if cell.cardContentView doesn’t already exist

  • getCardControllerCount()

    Number of cardControllers

    cardController from cardControllers model
